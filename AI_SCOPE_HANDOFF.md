# AI Scope Handoff

## 1. What this app is

Radio Presto is a single-page React frontend prototype for a futuristic, AI-powered radio/podcast platform. It presents a mobile-first UI where users can discover audio content by category, create personalized broadcasts by entering a topic, and control a now-playing player. The entire app is a visual mock: there is no real audio, no backend, and no live AI calls. It was generated by Google AI Studio and has not been extended since initial generation.

## 3. Functional scope discovered in code

**Working user flows:**
1. Navigate between Discover / Create / Active / Profile tabs
2. Open and close Settings and Nodes overlays
3. Follow/unfollow a station node or news category → generates in-app notification
4. Open notification dropdown → "Manage Follow" toggles follow state
5. Search the Discover screen → filters 12 hardcoded topic strings
6. Click a news card category → confirmation modal to add it as a new node
7. Manage nodes (Nodes screen) → drag-to-reorder, delete
8. Enter a topic on Create (text or voice) → 3-second loading screen → lands on Active tab
9. Play/pause, scrub, skip ±15s, skip next/prev on Active screen
10. View and reorder/delete hardcoded queue items on Active screen
11. Save the current (hardcoded) broadcast title → appears in Profile's saved list
12. Upload a profile avatar via file picker (FileReader, stored as base64)
13. Toggle dark/light theme (CSS custom properties on `<body>`)
14. Toggle haptic feedback (`navigator.vibrate`)

## 4. Screen-by-screen breakdown

### DiscoverScreen
- **Route/key:** `activeTab === "discover"`
- **Purpose:** Browse stations and trending news; search content; add nodes
- **Key UI sections:** Search bar with autocomplete; "My Stations" horizontal scroll (4 station cards); "Trending Global News" (3 news cards: 1 large, 2 small); "Add Node" confirmation modal
- **Data source:** All static — 4 hardcoded stations, 3 hardcoded news items, 12 hardcoded search suggestion strings
- **State used:** `searchQuery`, `showSuggestions`, `confirmNode` (local); `followedCategories`, `followedNodes` (from App)
- **User actions:** Search; follow/unfollow stations and categories; click category badge → "Add Node" modal
- **Limitations:** No real search; news content is static; "Tune In" / "Listen Now" buttons do nothing

### CreateScreen
- **Route/key:** `activeTab === "create"`
- **Purpose:** Input a topic (text or voice) to trigger broadcast "research"
- **Key UI sections:** Mic button (voice input); text input field; 4 quick-topic chips
- **Data source:** User input only
- **State used:** `isListening`, `inputValue` (local); Web Speech API via `recognitionRef`
- **User actions:** Type topic; use voice input; submit via Enter / arrow button / chip click
- **Limitations:** Submitting triggers only a 3-second timeout — no real AI call. Voice auto-submits on final speech result.

### ResearchingScreen
- **Route/key:** `isResearching === true` (overlays all tabs)
- **Purpose:** Loading state while "research" runs
- **Key UI sections:** Animated concentric rings; topic name; fake progress indicators; animated progress bar
- **Data source:** `researchTopic` prop (user-entered string)
- **State used:** None local
- **User actions:** None — auto-navigates to Active after 3s
- **Limitations:** Progress indicators are purely decorative; "Scanning news sources 100%" is hardcoded

### ActiveScreen
- **Route/key:** `activeTab === "active"`
- **Purpose:** Now-playing player for the current broadcast
- **Key UI sections:** Large artwork image; broadcast metadata; progress bar + playback controls; Save / Queue / Sources buttons; animated waveform; drag-reorderable queue panel; clear-queue confirmation modal
- **Data source:** Entirely hardcoded (`currentBroadcastTitle = "Neural Horizons: Part 04"`, artwork URLs, initial queue of 3 items)
- **State used:** `isPlaying`, `currentTime`, `showQueue`, `showClearConfirm`, `queue`, `showShareToast` (all local)
- **User actions:** Play/pause; scrub progress bar; skip ±15s; skip next (removes first queue item, resets time) / prev (resets time); remove/reorder queue items; clear queue; save broadcast title; open queue panel; share (shows toast only); open Sources (no action)
- **Limitations:** No real audio. "Sources" button has no handler. Share copies nothing. Skipping tracks only resets the timer. Content never changes based on the researched topic.

### ProfileScreen
- **Route/key:** `activeTab === "profile"`
- **Purpose:** User profile with saved broadcasts and activity history
- **Key UI sections:** Avatar (uploadable); username + level; stats grid; saved broadcasts list; hardcoded recent transmissions list
- **Data source:** `savedBroadcasts` from App (dynamic); username "Commander Peppino", stats (128/450/12), recent transmissions list are all hardcoded
- **State used:** `avatarUrl`, `savedBroadcasts` (from App); `fileInputRef` (local)
- **User actions:** Click avatar → file picker → uploads image (FileReader, base64); view saved broadcasts
- **Limitations:** Clicking saved broadcasts or recent transmissions items does nothing. Stats are fake.

### NodesScreen
- **Route/key:** `showNodes === true` (overlays all tabs)
- **Purpose:** Manage transmission nodes — reorder and delete
- **Key UI sections:** Back button; drag-reorderable node list with status indicators; empty state
- **Data source:** `nodes` array from App (starts with 4 hardcoded nodes; can be added/deleted)
- **State used:** `nodes`, `setNodes` (from App via `onReorder`)
- **User actions:** Drag to reorder; delete individual nodes; back
- **Limitations:** Node icons are resolved from a hardcoded `iconMap`; new nodes added from DiscoverScreen always use the `Activity` icon

### SettingsScreen
- **Route/key:** `showSettings === true` (overlays all tabs)
- **Purpose:** App settings — appearance, account, music services, privacy
- **Key UI sections:** Back button; theme toggle; haptic toggle; Sign in with Google/Apple; Connect Spotify/Apple Music; Personalized Frequency toggle; Sign Out button
- **Data source:** `theme`, `hapticEnabled` from App
- **State used:** `theme`, `hapticEnabled` (from App)
- **User actions:** Toggle theme; toggle haptic feedback; all other buttons are non-functional
- **Limitations:** Auth, music services, privacy toggle, and sign-out are all visual only. Privacy toggle has no state — always renders as "on".

## 5. Architecture and technical design

### App structure
Single-page application. `src/main.tsx` → `<App />`. All shared state lives in `App.tsx` and is passed down as props. No state management library.

### Routing/navigation
No router. Three boolean flags (`showSettings`, `showNodes`, `isResearching`) act as modal overlays. `activeTab: Tab` ("discover" | "create" | "active" | "profile") controls the main content. `AnimatePresence` + `motion.div` provide animated transitions.

### Component organization
- `src/App.tsx` — root, state owner, orchestrator
- `src/components/UI.tsx` — `TopNav`, `BottomNav`, `StationCard`, `NewsCard` (layout and shared display components)
- `src/components/Screens.tsx` — all 7 screen components in one file (~1250 lines)

### State management
Pure React `useState` + `useEffect`. No context, no external store. State is passed via props; callbacks passed up via handler props (e.g., `onStartResearch`, `onSave`, `onFollowNode`).

### Data handling
No API calls anywhere. All data is either:
- Hardcoded inline in JSX/component body
- Local `useState` initialized with hardcoded values
- User input (search query, avatar image, topic text)

### Styling system
Tailwind CSS v4 (via `@tailwindcss/vite` plugin). Theme colors defined as CSS custom properties in `:root` and `.light` blocks in `src/index.css`. Tailwind maps these via `@theme { --color-* }`. Custom utility classes: `.glass-panel`, `.text-glow`, `.neon-glow`, `.atmospheric-bg`, `.no-scrollbar`. Fonts: "Space Grotesk" (headlines), "Inter" (body) — loaded from CDN (not bundled; I cannot confirm the CDN source from the code as no `<link>` tag is visible in `index.html`).

### Key dependencies
- `motion/react` — `AnimatePresence`, `motion.div`, `Reorder.Group/Item` for drag-and-drop
- `lucide-react` — all icons
- `@google/genai` — installed but **never imported**
- `express`, `dotenv` — installed but **no server file exists**

## 7. Gaps, risks, and inconsistencies

1. **`@google/genai` is installed but never used.** The Gemini API key is wired through `vite.config.ts` but no source file imports the SDK. The core value proposition of the app (AI-generated broadcast content) is entirely absent.

2. **`express` and `dotenv` are in `dependencies` (not `devDependencies`), but no server exists.** This inflates the production bundle if not tree-shaken, and is misleading scaffolding.

3. **`triggerHaptic` is copy-pasted into 6+ locations** (`TopNav`, `BottomNav`, `StationCard`, `NewsCard`, `DiscoverScreen`, `ProfileScreen`) as an identical inline function. The real implementation with the enabled/disabled flag lives in `App.tsx` and is passed as a prop to `ActiveScreen` only. The duplicated versions always fire regardless of the user's haptic setting.

4. **Notifications are never marked as read.** The `read: boolean` field is always initialized to `false` and never flipped to `true`. The badge count always shows all notifications as unread.

5. **`jumpToNext` does not load the next broadcast.** It resets `currentTime` to 0 and removes the first queue item, but `currentBroadcastTitle` and artwork remain hardcoded. The active content never actually changes.

6. **The Share button claims to copy a link but calls no clipboard API.** `navigator.clipboard.writeText()` is never called.

7. **The privacy toggle in Settings has no backing state.** It always renders as toggled on (`right-1` thumb position), with no `onClick` handler.

8. **All profile stats are fiction.** "128 Broadcasts", "450 Hours", "12 Nodes" are hardcoded and bear no relationship to actual user behavior.

9. **No error handling anywhere.** Web Speech API failures, file reader errors, and any future API calls have zero error handling.

10. **Font loading is not visible in the bundled code.** The CSS references "Space Grotesk" and "Inter" fonts but no `@font-face` or CDN `<link>` is visible in `index.html` or `index.css`. Fonts may be falling back to system fonts silently.

## 8. Improvement opportunities (prioritized)

1. **Integrate Gemini API** in the Create flow — call `@google/genai` to generate actual broadcast content from the user's topic. This is the app's core premise and the SDK is already installed.
2. **Add real audio playback** — use the Web Audio API or `<audio>` element with a TTS or stream URL. Without this the player controls are meaningless.
3. **Fix the haptic bug** — move `triggerHaptic` into a React context or pass it from `App.tsx` so the enabled/disabled setting is respected everywhere.
4. **Mark notifications as read** — set `read: true` when the notification dropdown is opened, or when a notification is clicked.
5. **Add persistence** — `localStorage` for user preferences (theme, haptic, follows) would work without a backend and would survive page refresh.
6. **Wire up the Share button** — call `navigator.clipboard.writeText(window.location.href)` or use the Web Share API.
7. **Add a real backend** — `express` is already installed. A minimal server could handle auth, user data, and AI proxy calls (to avoid exposing the Gemini key client-side).
8. **Remove dead dependencies** — move or remove `express` and `dotenv` from `dependencies` if no server is being added.
9. **Split `Screens.tsx`** — the file is ~1250 lines. Each screen should be its own module.
10. **Add font loading** — confirm fonts load correctly and add explicit `@font-face` or Google Fonts `<link>` tags if missing.

## 10. Reusable AI context summary

Radio Presto is a React 19 + TypeScript frontend-only prototype for an AI radio platform. It is a tab-based SPA (no router) with state managed in a single root `App.tsx` and all UI split across `UI.tsx` (nav/cards) and `Screens.tsx` (screens). The design is futuristic/dark with a CSS custom property theme system. There is **no real audio, no AI integration, and no backend** — the app is a polished visual mockup. Key next-development tasks are: integrating the already-installed `@google/genai` SDK into the Create → Research flow, wiring up an audio player, and adding persistence. The codebase has a haptic state bug (duplicated local functions ignore the global enabled flag) and several non-functional UI buttons (Share, Auth, Music services, Privacy toggle). All broadcast content is hardcoded.
